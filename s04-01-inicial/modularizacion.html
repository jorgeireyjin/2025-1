Modularización

Antecedentes

JavaScript se ha estandarizado en lo que se conoce como ES6 (ECMAScript 2015) Antes de eso no existia un estandar para generar modulos. Lo existente se convirtio en estandar de-facto (CommonJS, RequireJS, AMD).

Problema

Al programar JS se dividia el código en archivos que se concatenaban al generar la aplicación para producir un único archivo fuente. Esta práctica, podia generar varios problemas: colisiones de nombres entre archivos (por ejemplo: una misma variable global declarada en archivos distintos). multiples referencias al mismo archivo podían resultar en el mismo código siendo ejecutado varias veces

Solucion:

Se introduce el concepto de modulo como una unidad aislada:
  1. Cada modulo posee su propio scope lo que previene las colisiones de nombre.
  2. Cada modulo es una clase de tipo singleton que puede ser importado varias veces en distintos momentos.



Algunas formas de modularizar en JS:

  1. Expresión de función inmediatamente invocada (IIFE)
  2. CommonJS (CJS)
  3. Definición de módulo asíncrono (AMD)
  4. Definición de módulo universal (UMD)
  5. Módulos ECMAScript (ESM)
